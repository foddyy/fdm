# Face Distance Monitor - Android 应用

一个后台运行的 Android 应用，使用前置摄像头实时监测用户面部与手机屏幕的距离。
当检测到面部距离屏幕小于 30cm 时，会在屏幕上显示四边闪烁红光的警示效果。

## 📱 功能特性

- **实时距离监测**：使用 CameraX + ML Kit 人脸检测技术
- **智能校准**：启动时进行双眼间距校准，建立距离基准
- **后台运行**：以前台服务方式持续运行，不依赖应用在前台
- **闪烁警示**：面部过近时，屏幕四边红色闪烁提醒
- **悬浮窗显示**：即使应用退到后台也能看到警示效果

## 🔧 使用方法

### 1. 首次使用

```bash
# 克隆项目
git clone <repository-url>
cd FaceDistanceMonitor

# 使用 Android Studio 打开项目
# 或者通过命令行构建
./gradlew assembleDebug
```

### 2. 安装到设备

```bash
# 安装 APK
adb install app/build/outputs/apk/debug/app-debug.apk

# 或者通过 Android Studio Run 按钮直接部署
```

### 3. 校准步骤

1. 打开应用，自动跳转到校准界面
2. 将手机放在正常阅读距离（约 30-40cm）
3. 正对前置摄像头，确保面部清晰可见
4. 等待 2 秒自动采集数据
5. 校准成功后自动返回主界面

### 4. 开始监控

1. 在主界面点击"开始监控"
2. 授予悬浮窗权限（如果需要）
3. 应用进入后台服务模式
4. 当面部距离 < 30cm 时，屏幕四边闪烁红光

### 距离阈值

在 `DistanceMonitorService.kt` 中修改：

```kotlin
const val THRESHOLD_DISTANCE_CM = 30  // 修改为你想要的阈值
const val NORMAL_READING_DISTANCE_CM = 35  // 校准时的正常距离
```

### 闪烁频率

```kotlin
const val BLINK_INTERVAL_MS = 200  // 闪烁间隔（毫秒）
const val BLINK_DURATION_MS = 150  // 每次闪烁持续时间（毫秒）
```

## 🔐 所需权限

- `CAMERA` - 访问前置摄像头
- `FOREGROUND_SERVICE` - 运行后台服务
- `FOREGROUND_SERVICE_CAMERA` - 后台使用摄像头
- `POST_NOTIFICATIONS` - 显示通知（Android 13+）
- `SYSTEM_ALERT_WINDOW` - 显示悬浮窗警示

## 📝 工作原理

### 距离估算算法

```
1. 校准阶段：
   - 用户保持正常阅读距离面对摄像头
   - ML Kit 检测左右眼关键点
   - 计算双眼之间的像素距离 → baselineEyeDistancePx
   
2. 监测阶段：
   - 实时检测双眼像素距离 → currentEyeDistancePx
   - 使用相似三角形原理估算距离：
     estimatedDistance = normalDistance × (baselinePx / currentPx)
   
3. 预警触发：
   - 如果 estimatedDistance < 30cm → 启动红色闪烁警示
   - 如果 estimatedDistance >= 30cm → 关闭警示
```

### 为什么使用双眼距离而不是面部大小？

| 指标 | 稳定性 | 精度 | 抗干扰能力 |
|------|--------|------|-----------|
| 双眼距离 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 面部大小 | ⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ |
| 面部面积 | ⭐⭐ | ⭐⭐ | ⭐⭐ |

**双眼距离的优势：**
- 不受头部倾斜角度影响
- 不受表情变化影响
- 物理距离恒定，更适合做参照
- 检测精度高，ML Kit 支持亚像素级定位

## 🚨 注意事项

1. **精度限制**：这是基于视觉的估算方法，实际精度受以下因素影响：
   - 光线条件
   - 用户面部朝向
   - 摄像头分辨率
   - 个体面部特征差异

2. **电量消耗**：持续使用摄像头和 ML Kit 会消耗较多电量，建议：
   - 仅在需要时开启监控
   - 降低分析帧率（可通过调整 CameraX 配置）

3. **隐私保护**：
   - 所有图像处理在本地完成
   - 不上传任何人脸数据
   - 校准数据存储在本地 SharedPreferences

4. **兼容性**：
   - 最低支持 Android 8.0 (API 26)
   - 建议使用 Android 10+ 以获得最佳悬浮窗支持

## 🛠️ 故障排除

### 问题：校准失败
- **原因**：光线太暗或面部未被检测到
- **解决**：确保光线充足，正对摄像头

### 问题：警示不显示
- **原因**：悬浮窗权限未授予
- **解决**：设置 → 应用 → Face Distance Monitor → 允许显示在其他应用上层

### 问题：距离估算不准
- **原因**：校准距离与实际使用距离差异过大
- **解决**：重新校准，确保校准时的距离接近日常使用距离
